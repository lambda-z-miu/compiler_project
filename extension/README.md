# my-Chem Renderer Extension

This extension replaces HTML tags like:

```html
<my-Chem>(0)NC=OC,(0);</my-Chem>
```

with SVG generated by the C++ compiler backend.

## Install

### Localhost Service Mode

This is the recommended mode for Windows Chrome/Edge talking to the WSL backend.

1. Build the HTTP service:

   ```bash
   cd /home/heziy/compiler_project
   cmake -S . -B . -DREQUIRE_RDKIT=ON
   cmake --build . --target chem_http_server
   ```

2. Start the service:

   ```bash
   /home/heziy/compiler_project/chem_http_server
   ```

   It listens on:

   ```txt
   http://127.0.0.1:17654
   ```

3. Load the extension from:

   ```txt
   /home/heziy/compiler_project/extension
   ```

4. Refresh any page containing `<my-Chem>` tags.

### Native Messaging Mode

Native messaging is kept as a fallback for Linux Chrome/Edge.

1. Build the native host:

   ```bash
   cd /home/heziy/compiler_project
   cmake -S . -B . -DREQUIRE_RDKIT=ON
   cmake --build . --target chem_native_host
   ```

2. Open Chrome/Edge extension management:

   - Chrome: `chrome://extensions`
   - Edge: `edge://extensions`

3. Enable developer mode, load unpacked extension, and choose:

   ```txt
   /home/heziy/compiler_project/extension
   ```

4. Copy the extension ID shown by the browser.

5. Install the native messaging manifest:

   ```bash
   /home/heziy/compiler_project/tools/install_native_host.sh EXTENSION_ID both
   ```

   Use `chrome` or `edge` instead of `both` if you only need one browser.

6. Reload the extension and refresh any page containing `<my-Chem>` tags.

## Note

Native messaging manifests are browser-profile local. If the extension ID changes,
rerun `tools/install_native_host.sh` with the new ID.
